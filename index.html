Welcome to the HRI Public Deployment Checklist!

We created this tool to help researchers identify potential blind spots in their deployment and make the deployment smoother. We hope it covers all aspects of the deployment, from the robot's technical details to research procedures, and to public and site interaction.

If you have already been using the checklist, welcome back. Use Upload JSON to load your previous version and continue editing from where you left off.

Authors anonymous due to double blind submission process. Questions and feedback can be addressed anonymously to hriiprobot@gmail.com.

Checklist Contribution Guide

Use this flow when you want to add or revise checklist items or descriptions.

GitHub Website Workflow

1. Open repository
2. Create branch
3. Edit checklist locally in this site
4. Download edited JSON
5. Upload file in GitHub
  1. Open the repository and go to the branches page.
  2. Create a new branch from the default branch.
  3. Return to this website, turn on Edit Mode, open a card, revise the title/items/description, and save the card locally.
  4. Use Download Edited JSON to save the updated latest_cards.json.
  5. In GitHub, open your new branch, choose to upload or replace latest_cards.json, and commit that file to your branch.
  6. Add a new README.md in that branch with a description of the study, what happened in deployment, why it matters, and any suggested checklist additions.
  7. Open a pull request from your branch when ready.

Terminal Workflow

terminal
git clone https://github.com/publicrobot/publicrobot.github.io
cd publicrobot.github.io
git checkout -b my-checklist-update
# replace latest_cards.json with your downloaded file
git add latest_cards.json
git commit -m "Update checklist content"
git push origin my-checklist-update
  1. Clone the repository and create a new branch with git checkout -b <branch-name>.
  2. Edit content in this website and download the new latest_cards.json.
  3. Replace the repository’s existing latest_cards.json with the downloaded file.
  4. Create or update a README.md in your branch with a description of the study context, what happened, why it matters, and suggested checklist implications.
  5. Run git add latest_cards.json README.md, commit your change, and push the branch.
  6. Open a pull request on GitHub from that branch.

Broader Suggestions Guide

Use this path for broader feedback, missing topics, unclear wording, or discussion points that should become a GitHub issue.

GitHub Website Workflow

1. Open issues tab
2. Start new issue
3. Add title + context
4. Describe suggestion
5. Submit issue
  1. Open the repository issues page.
  2. Create a new issue.
  3. Use a short topic title, explain what is missing or unclear, and describe the suggested improvement.
  4. Include the relevant phase and column if you know them: Before, During, or After and Robot, Human, or Public.
  5. Submit the issue for discussion.

Terminal Workflow

terminal
gh auth login
gh repo set-default publicrobot/publicrobot.github.io
gh issue create \
  --title "Missing topic: accessibility logistics" \
  --body "Describe the gap and suggested improvement."
  1. Install the GitHub CLI and authenticate with gh auth login.
  2. Set the repository with gh repo set-default publicrobot/publicrobot.github.io.
  3. Create an issue with gh issue create, including the topic, the gap, and the suggested improvement.
  4. Review the created issue in GitHub and continue discussion there.
